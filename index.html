Git for Beginners

Everything you need to start tracking and sharing your code.

Step 1

What is Git?

Git is a version control system. It tracks every change you make to your files so you can go back in time, collaborate with others, and never lose your work.

Think of it like save points in a video game — you create a save point whenever you want, and you can always reload one if something goes wrong.

Step 2

Start a new project

Navigate to your project folder in the terminal, then run:

git init

This creates a hidden .git folder that Git uses to track everything. You only do this once per project.

Step 3

Check what has changed

Before saving anything, see the current state of your files:

git status

Files in red are changed but not yet staged. Files in green are staged and ready to commit.

Step 4

Stage your changes

Staging means telling Git which changes to include in your next save point. Add a specific file:

git add index.html

Or stage everything at once:

git add .
Tip: Use git add . carefully — it stages all files including ones you may not want to commit.
Step 5

Commit your changes

A commit is your save point. Always include a short message describing what you changed:

git commit -m "add homepage"

Good commit messages are short and say what changed, not how. Examples:

git commit -m "fix broken login button" git commit -m "add contact page" git commit -m "update nav colours"
Step 6

Connect to GitHub

GitHub is a website that stores your code online. First create a new repository on GitHub, then link it to your local project:

git remote add origin git@github.com:yourusername/your-repo.git

You only need to do this once. Replace the URL with your own repository URL.

Step 7

Push your code to GitHub

Pushing sends your commits to GitHub so they are stored online and others can see them:

git push -u origin main

After the first push, you can just use:

git push
Step 8

Your everyday workflow

Once your project is set up, this is all you do each time you make changes:

git add . git commit -m "describe what you changed" git push

That is it. Three commands to save and back up your work.

Quick Reference

git init Start a new Git repository
git status See what has changed
git add . Stage all changes
git commit -m "message" Save a snapshot with a message
git push Upload commits to GitHub
git pull Download latest changes from GitHub
git log --oneline See a history of your commits
git diff See exactly what lines changed
